# AZ-900 Cheat Sheet – Last Minute Review

## ☁️ Cloud Basics

* OpEx = pay-as-you-go
* CapEx = upfront investment
* Cloud = scalable + elastic

---

## 🏗️ Infrastructure

* Region = multiple datacenters
* Availability Zone = separate datacenter within a region

---

## ⚙️ Compute

| Service     | Use Case         |
| ----------- | ---------------- |
| VM          | Full control     |
| Containers  | Lightweight apps |
| Functions   | Event-driven     |
| App Service | Web apps & APIs  |

---

## 🌐 Networking

| Service      | Purpose                    |
| ------------ | -------------------------- |
| VNet         | Private network            |
| VPN          | Secure internet connection |
| ExpressRoute | Private connection         |
| DNS          | Name resolution            |
| NSG          | Traffic filtering          |

---

## 💾 Storage

| Type  | Use Case      |
| ----- | ------------- |
| Blob  | Files, images |
| File  | Shared drives |
| Queue | Messaging     |
| Table | NoSQL         |

---

## 🔁 Redundancy

* LRS = single region
* ZRS = multiple zones
* GRS = geo-redundant
* GZRS = zones + geo

---

## 💰 Cost Management

* Pricing Calculator = estimate
* Cost Management = analyze
* Tags = organize costs

---

## 🛡️ Governance

* Azure Policy = enforce rules
* Resource Locks = prevent deletion
* Purview = data governance

---

## 📊 Monitoring

| Tool            | Purpose         |
| --------------- | --------------- |
| Azure Monitor   | metrics + logs  |
| Advisor         | recommendations |
| Service Health  | Azure outages   |
| Resource Health | resource status |

---

## 🛠️ Tools

* Portal = GUI
* CLI / PowerShell = automation
* ARM = infrastructure as code
* Arc = hybrid

---

## ⚠️ Exam Traps

* Service Health ≠ Resource Health
* NSG ≠ Azure Firewall
* IaaS ≠ PaaS ≠ SaaS
* Availability Zone ≠ Region

---

## 🚀 Final Tip

Focus on:

* Differences between services
* Use cases
* Not implementation details
